{{- else if eq .Values.frontend.service.type "LoadBalancer" }}

To access the dashboard, get the external IP of the frontend service:

  export SERVICE_IP=$(kubectl get svc --namespace {{ .Release.Namespace }} {{ include "rocketmq-dashboard.fullname" . }}-frontend -o jsonpath='{.status.loadBalancer.ingress[0].ip}')
  echo "RocketMQ Dashboard: http://$SERVICE_IP"

NOTE: It may take a few minutes for the LoadBalancer IP to be available.
      You can watch the status with:

  kubectl get svc --namespace {{ .Release.Namespace }} -w {{ include "rocketmq-dashboard.fullname" . }}-frontend

{{- else if eq .Values.frontend.service.type "ClusterIP" }}

To access the dashboard, forward the frontend port to your local machine:

  kubectl --namespace {{ .Release.Namespace }} port-forward svc/{{ include "rocketmq-dashboard.fullname" . }}-frontend 8080:{{ .Values.frontend.service.port }}

Then open: http://localhost:8080

{{- end }}
